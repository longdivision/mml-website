## Why React?

While the MML web client functions effectively with HTML and JavaScript, integrating it with React
offers distinct advantages for dynamic and complex applications. React's component-based
architecture promotes modular design and reusability, aligning well with MML's modular nature. It
offers efficient state management, allowing for responsive UIs with real-time updates and smooth
user experiences. The thriving React community provides a wealth of resources and solutions, while
its testability and team collaboration benefits contribute to maintainable code. While both
approaches are valid, using React enhances development efficiency, UI consistency, and application
performance when working with MML.

To start working with react, we recommend using our
[React starter project](https://github.com/mml-io/mml-react-starter-project) as a start. We will

## Reusability with React

React's component-based architecture promotes reusability and modularity. This aligns well with
MML's modular nature, allowing for the creation of reusable components that can be easily integrated
into other projects. For example, we can create a `MMLScene` component that renders a scene with a
light and a plane that accepts children to render other elements.

```jsx
// MMLScene.js
import React from "react";
import Light from "./Light";

export default function MMLScene({ children }) {
  return (
    <m-group>
      <Light />
      <m-plane width={100} height={100} rx={-90} color="green" />
      {children}
    </m-group>
  );
}
```

We can then use this component in our `App` component to render the current scene and reuse it in
other parts of our application.

## MML Events in React

Some MML elements have events that can be used to trigger actions in the scene. Unfortunately, React doesn't allow
us to pass those events directly to the elements. We need to use a few workarounds to be able to use them. We are going
to see a few common cases in the sections below. These examples require a user avatar in the scene. We recommend
using [MML React Space](https://github.com/mml-io/mml-react-space) to play around with these examples.

## Collision events
Collision events are triggered when a user collides with an MML element. Let's create an example that uses all the
collision events available in MML. We are going to use a `m-cube` as a platform and a `m-cube` as a panel to display
the position of the users that are currently in the scene. We are going to use the `collisionstart` event to add the users to the
panel and the `collisionend` event to remove the users from the panel. We are also going to use the `collisionmove` event
to update the position of the user in the panel.

Let's start with the basic scene structure.

```jsx
export default function CollisionReactExample() {
  return (
    <m-group>
      <m-cube id="users-panel" height="6" width="12" depth="0.1" y="4" z="-2">
        <m-group rx="90" id="user-presence-holder"></m-group>
      </m-cube>
      <m-cube
        id="platform"
        y="0.05"
        height="0.1"
        width="6"
        depth="3"
        color="green"
        collision-interval="100"
      ></m-cube>
    </m-group>
  );
}
```

![collision-react-example](/images/guides/mml-with-react/react-collision-start.jpg)

We can see the panel and the platform in the image above. Let's add the logic to handle the `collisionstart` events.

```jsx
import React, { useEffect, useRef, useState } from "react";
export default function CollisionReactExample() {
  const platformRef = useRef(null);
  const [users, setUsers] = useState(new Map());

  function handleCollisionstart(e) {
    const { connectionId, position } = e.detail;

    const newUsers = createUser(connectionId, position);
    setUsers(newUsers);
  }

  function createUser(connectionId, position) {
    const user = users.get(connectionId);

    if (user) {
      user.position = position;
      return user;
    }

    const newUser = {
      color: `#${Math.floor(Math.random() * 0xffffff).toString(16)}`,
      position,
    };
    return new Map(users).set(connectionId, newUser);
  }

  useEffect(() => {
    const platform = platformRef.current;

    platform.addEventListener("collisionstart", handleCollisionstart);

    return () => {
      platform.removeEventListener("collisionstart", handleCollisionstart);
    };
  }, []);

  return (
    <m-group>
      <m-cube id="users-panel" height="6" width="12" depth="0.1" y="4" z="-2">
        <m-group rx="90" id="user-presence-holder">
          {[...users.entries()].map(([connectionId, cubeData]) => (
            <m-cube
              key={connectionId}
              collide={false}
              width="0.25"
              height="0.25"
              depth="0.25"
              y={0.25}
              position={cubeData.position}
              color={cubeData.color}
            />
          ))}
        </m-group>
      </m-cube>
      <m-cube
        id="platform"
        ref={platformRef}
        y="0.05"
        height="0.1"
        width="6"
        depth="3"
        color="green"
        collision-interval="100"
      ></m-cube>
    </m-group>
  );
}
```

In the example above we are using a `Map` to store the users in the scene. We are using the `connectionId` as the key
and an object with the `color` and `position` of the user as the value. We are using the `createUsers` function to
create a user if it doesn't exist. We are using React `ref` to get the `platform`
element and then we are using `useEffect` to add the event listener when the component is mounted.
We are also using the `useEffect` hook to remove the event listener when the component is unmounted.
Stepping onto the platform will trigger the `collisionstart` event and add the user to the `users` map. The state update
will trigger a re-render and the user will be added to the panel.

Let's handled the `collisionmove` event now.

```jsx
export default function CollisionReactExample() {
  const platformRef = useRef(null);
  const [users, setUsers] = useState(new Map());

  function setCubePosition(connectionId, position) {
    // To make sure our state is updated correctly we need to put our logic inside the setUsers callback
    setUsers((oldUsers) => {
      const user = oldUsers.get(connectionId);

      if (!user) {
        return createUser(connectionId, position);
      }

      // Update the user's position
      const newUser = { ...user };
      newUser.position = {
        x: position.x * 2,
        z: position.z * 2,
      };

      // Update the users map
      return new Map(oldUsers).set(connectionId, newUser);
    });
  }

  function handleCollisionstart(e) {
    const { connectionId, position } = e.detail;

    const newUsers = createUser(connectionId, position);
    setUsers(newUsers);
  }

  function handleCollisionmove(e) {
    const { connectionId, position } = event.detail;
    setCubePosition(connectionId, position);
  }

  function createUser(connectionId, position) {
    const user = users.get(connectionId);

    if (user) {
      user.position = position;
      return user;
    }

    const newUser = {
      color: `#${Math.floor(Math.random() * 0xffffff).toString(16)}`,
      position,
    };
    return new Map(users).set(connectionId, newUser);
  }

  useEffect(() => {
    const platform = platformRef.current;

    platform.addEventListener("collisionstart", handleCollisionstart);
    platform.addEventListener("collisionmove", handleCollisionmove);

    return () => {
      platform.removeEventListener("collisionstart", handleCollisionstart);
      platform.removeEventListener("collisionmove", handleCollisionmove);
    };
  }, []);

  return (
    <m-group>
      <m-cube id="users-panel" height="6" width="12" depth="0.1" y="4" z="-2">
        <m-group rx="90" id="user-presence-holder">
          {[...users.entries()].map(([connectionId, cubeData]) => (
            <m-cube
              key={connectionId}
              collide={false}
              width="0.25"
              height="0.25"
              depth="0.25"
              y={0.25}
              x={cubeData.position.x}
              z={cubeData.position.z}
              color={cubeData.color}
            />
          ))}
        </m-group>
      </m-cube>
      <m-cube
        id="platform"
        ref={platformRef}
        y="0.05"

        depth="3"
        color="green"

      ></m-cube>
    </m-group>
  );
}
```

We are using the `setCubePosition` function to update the position of the user in the panel. We are using the
`setUsers` callback to make sure our state is updated correctly. We are also multiplying the `x` and `z` position by 2
to show the user position in the panel correctly since it's twice the size of the platform.

Finally, let's handle users leaving the platform and disconnections.

```jsx
export default function CollisionReactExample() {
  const platformRef = useRef(null);
  const [users, setUsers] = useState(new Map());

  function setCubePosition(connectionId, position) {
    // Get the user from the users map
    setUsers((oldUsers) => {
      const user = oldUsers.get(connectionId);

      // If the user doesn't exist, create it
      if (!user) {
        return createUser(connectionId, position);
      }

      // Update the user's position
      const newUser = { ...user };
      newUser.position = {
        x: position.x * 2,
        z: position.z * 2,
      };

      // Update the users map
      return new Map(oldUsers).set(connectionId, newUser);
    });
  }

  function handleCollisionstart(e) {
    const { connectionId, position } = e.detail;

    const newUsers = createUser(connectionId, position);
    setUsers(newUsers);
  }

  function handleCollisionmove(e) {
    const { connectionId, position } = event.detail;
    setCubePosition(connectionId, position);
  }

  function handleCollisionend(e) {
    const { connectionId } = e.detail;
    clearUser(connectionId);
  }

  function createUser(connectionId, position) {
    const user = users.get(connectionId);

    if (user) {
      user.position = position;
      return user;
    }

    const newUser = {
      color: `#${Math.floor(Math.random() * 0xffffff).toString(16)}`,
      position,
    };

    return new Map(users).set(connectionId, newUser);
  }

  function clearUser(connectionId) {
    setUsers((oldUsers) => {
      const user = oldUsers.get(connectionId);
      if (!user) return oldUsers;
      oldUsers.delete(connectionId);
      return new Map(oldUsers);
    });
  }

  useEffect(() => {
    const platform = platformRef.current;

    // When a user disconnects, their representation is cleared.
    window.addEventListener("disconnected", (event) => {
      const { connectionId } = event.detail;
      clearUser(connectionId);
    });

    platform.addEventListener("collisionstart", handleCollisionstart);
    platform.addEventListener("collisionmove", handleCollisionmove);
    platform.addEventListener("collisionend", handleCollisionend);

    return () => {
      platform.removeEventListener("collisionstart", handleCollisionstart);
      platform.removeEventListener("collisionmove", handleCollisionmove);
      platform.removeEventListener("collisionend", handleCollisionend);
    };
  }, []);

  return (
    <m-group>
      <m-cube id="users-panel" height="6" width="12" depth="0.1" y="4" z="-2">
        <m-group rx="90" id="user-presence-holder">
          {[...users.entries()].map(([connectionId, cubeData]) => (
            <m-cube
              key={connectionId}
              collide={false}
              width="0.25"
              height="0.25"
              depth="0.25"
              y={0.25}
              x={cubeData.position.x}
              z={cubeData.position.z}
              color={cubeData.color}
            />
          ))}
        </m-group>
      </m-cube>
      <m-cube
        id="platform"
        ref={platformRef}
        y="0.05"
        height="0.1"
        width="6"
        depth="3"
        color="green"
        collision-interval="100"
      ></m-cube>
    </m-group>
  );
}
```

We are using the `clearUser` function to remove the user from the `users` map. We are also listening for the
`disconnected` event to remove the user from the map when they disconnect. We are using the `setUsers` callback to
make sure our state is updated correctly.
